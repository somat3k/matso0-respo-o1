# Code of Conduct

## Purpose

This project aims to build serious financial and research infrastructure. Collaboration must be disciplined, evidence-based, respectful, and safe.

## Expected conduct

Contributors are expected to:
- communicate clearly and directly,
- critique designs, not people,
- document assumptions before implementation,
- identify risk, failure, and ambiguity early,
- avoid presenting speculation as fact,
- protect credentials, user funds, and confidential data.

## Unacceptable conduct

The following are unacceptable:
- harassment, intimidation, or abuse,
- reckless handling of secrets, wallets, signing material, or customer data,
- publishing unverified financial claims as if they were validated,
- bypassing review for critical runtime, trading, or security changes,
- shipping code that intentionally disables controls without explicit approval.

## Enforcement

Maintainers may restrict or remove access for conduct that puts the project, users, or infrastructure at risk.
